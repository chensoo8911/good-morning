什麼是「空白狀態」（Empty State）？

當使用者第一次打開 App，或是購物車裡什麼東西都沒有的時候，那個空空的畫面就叫做「空白狀態」。對新手設計師來說，這常常是被忘記設計的地方！一個好的空白狀態不應該只是白茫茫一片，而是要趁這個機會「引導」使用者去採取行動，例如放個可愛的插圖配上一句「這裡還空空的，趕快去挑選喜歡的商品吧！」並加上一顆明顯的按鈕。這能大幅減少使用者的困惑感，讓體驗變得更友善。

什麼是「麵包屑導覽」（Breadcrumbs）？

麵包屑導覽就像是童話故事裡用來認路的小石子。它通常出現在網頁的左上方（例如：首頁 > 產品分類 > 筆記型電腦），清楚告訴使用者「你現在在哪個位置」，並讓他們能點擊任何一個上一層快速退回。對剛入門的設計師來說，當你在設計層級很深、內容龐雜的網站（如大型電商或新聞平台）時，加上麵包屑可以大大降低使用者迷路的焦慮感，是一個看似不起眼卻極為實用的好體驗設計！

什麼是「限色設計」（Limited Color Palette）？ 就是整個網站只用 2 種主色（通常一深一淺，像今天 IZANAMI 的近黑配米白、Glitch&Grit 的純黑配米白）撐起全站視覺。看起來簡單，其實是在考驗設計師的「紀律」——沒有多的顏色可以掩飾排版或動效的瑕疵，每個對比、留白、層次都要靠這兩色的深淺變化和字體、動態效果去撐出來。對新手來說，這是很好的練習方法：下次做作品集或練習作，試著逼自己只用兩色，會發現配色決策反而變輕鬆，畫面也更容易做出高級感。

【今日一學】

什麼是「雙鑽石設計流程」（Double Diamond）？ 這是設計圈最經典的方法論之一，長得像兩顆並排的菱形：第一顆鑽石先「發散」再「收斂」找出對的問題（Discover → Define），第二顆鑽石再「發散」再「收斂」找出對的解法（Develop → Deliver）。簡單說就是「先確定問題沒問錯，再動手解決」。今天【設計趨勢】那則新聞提到 AI 時代有人主張要用新框架取代它，但雙鑽石的核心精神——別急著畫圖、先想清楚問題——對新手來說仍是最基本也最重要的功課。

核心：不要收集功能要變習慣，不要當執行者要當調度 AI 的導演。L1 業餘愛好者（反向面試法＋自我審查）→ L2 常客（Projects、Master Prompt）→ L3 整合者（Connectors、Artifacts）→ L4 操作者・人在環中（技能化、排程協作）→ L5 建造者（Claude Code、Loops/Tools/Apps、/plan）→ L6 代理人編排者・人在環外（Orchestrator＋子代理人＋批判代理人）。落地建議：一項技巧連續用 30 天內化再升級。

4. Claude 的 .md 檔案家族

核心分野：README 寫給人看（交接用），CLAUDE.md 寫給 AI 看（指揮規則）。指令階層（越具體越優先）：managed policy > 全域 ~/.claude/CLAUDE.md > 專案根 CLAUDE.md > 子資料夾 CLAUDE.md > CLAUDE.local.md（本機不進 git）。.claude/commands/*.md＝手動觸發的斜線指令；skill 的 SKILL.md＝Claude 自己判斷觸發。@ 語法可在 CLAUDE.md 裡引入其他檔案內容。

5. Figma MCP 實用指令清單

5 個指令依實用度：get_design_context（抓完整設計脈絡，最常用）、get_screenshot（視覺截圖）、get_variable_defs（設計變數 tokens）、get_metadata（結構化基本資料，較輕量）、get_figjam（白板內容）。黃金組合：screenshot＋design_context＋variable_defs 三個搭起來轉程式碼。

7. Git remote／GitHub 帳號切換補充

remote＝git 幫本機資料夾記住的遠端網址書籤；一個資料夾可同時設兩個 remote（如 origin=個人、sv=公司），同一份檔案改一次可推去兩個不同 GitHub 帳號/repo。gh-sv／gh-me／gh-who 這些捷徑實際上是裝在 ~/.zshrc 裡的內容。

外觀先調（Theme、等寬字型、Opacity 微透明 85–95%）最有感。Agent 工作流賣點：Parallels/Monitor 多 agent 並排看進度、Composer+Prompt Queue、Send to Chat。Copy/Paste：Paste Protection（貼危險指令跳警告）建議留著。已知坑：關了 Copy on Select 卻好像還在複製 → 其實是 Session recovery 還原舊分頁狀態，需 ⌘Q 完全結束後重開才驗證得到。

9. ~/.zshrc 是什麼

終端機每次開啟自動讀取執行的個人設定檔，裝了自訂捷徑 alias（如小G）、環境變數、自訂函式。比喻：像 Illustrator 開啟時自動載入的工作區偏好設定。

10. 文件規範 vs 機制強制

同一件「要檢查」的事：寫進 SKILL.md＝文件規範（靠人/AI 遵守，可能被跳過）；寫進 release.sh＝機制強制（寫死在執行路徑，繞不過）。比喻：SKILL.md 像印刷 SOP 手冊（靠師傅記得翻），release.sh 像印刷機自動感測器。原則：漏掉會出事的檢查沉到機制裡，建議性判斷留文件即可。

11. 自動化工具三分類

Make.com/n8n（iPaaS 串 App 流程）、GitHub（VCS 存程式碼協作）、Google Antigravity（Agentic Development Platform，AI 代理寫程式）三者用途完全不同。Make 更視覺化簡單但只雲端非開源；n8n 更技術但開源可自架。自動化工具跑 agent：本質是流程確定性 vs 自主決策，n8n 內建 AI Agent 節點適合「有框架的 agent 工作流」，高度自主長鏈推理仍需專門框架。

12. 自動化的兩種型態

自動修正（唯一正解、靜默處理不問，如手機號碼強制排版）vs 自動偵測（需人決定、印 ⚠️ 警告逼流程停下問，如職稱中英混填）。核心洞察：腳本化的價值不一定是「以後都不用問」，而是消滅「漏問」的風險。
